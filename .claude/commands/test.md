description: 自动运行项目单元测试及 YAML 案例并修复失败项
---

# 1. 环境自检
检测项目语言并确定测试框架：
- Node.js: 查找 `package.json` (npm/yarn/pnpm test)
- Python: 查找 `requirements.txt` 或 `pyproject.toml` (pytest)
- Java: 查找 `pom.xml` 或 `build.gradle` (mvn/gradle test)

# 2. 执行测试
运行上述框架对应的单元测试命令。
搜索项目中所有 `**/*.yaml` 或 `**/*.yml` 测试案例，并调用相应的测试引擎执行。

# 3. 结果分析与修复
识别所有失败的测试用例及错误信息。
针对以下维度进行自动修复：
- 修改代码逻辑、修复断言预期。
- 更新配置文件或修复 YAML 测试数据。
- 安装缺失的依赖。

# 4. 迭代验证
重复“执行测试 -> 修复”流程，直到所有测试通过。
若遇到无法修复的问题，简要说明原因。

# 5. 约束
- 禁止生成任何格式的测试报告。
- 专注于代码修复，无需输出结果文档。
