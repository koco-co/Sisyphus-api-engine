## 编写规范

### 1. 关键字表格

每个功能文档必须包含完整的 YAML 关键字表格，包括：
- 关键字名称
- 数据类型
- 是否必填
- 默认值
- 功能描述

### 2. 代码示例

- 每个功能至少提供 3 个示例（基础、进阶、组合）
- 示例必须可直接运行
- 示例后需附运行结果或预期输出

### 3. 推荐组合

- 说明该功能与其他功能的最佳组合方式
- 提供组合使用的 YAML 示例

### 4. 链接规范

- 确保所有内部链接使用相对路径
- 链接到示例文件时使用 `../examples/` 路径
- 上/下一章链接放在文档末尾

### 5. 中文规范

- 使用规范的中文技术术语
- 专有名词首次出现时附带英文原名
- 代码注释使用中文
